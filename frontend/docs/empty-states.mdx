# Empty States in FreightDocs

Empty states are a critical part of the user experience. They provide feedback when no data is available and guide users on what actions they can take.

## EmptyState Component

The `EmptyState` component provides a consistent way to display empty states across the application.

### Usage

```tsx
import { EmptyState } from '@/components/freight/EmptyState';
import { FileText } from 'lucide-react';

// Basic usage
<EmptyState
  icon={FileText}
  title="No documents found"
  description="Try adjusting your filters or upload a new document"
/>

// With action button
<EmptyState
  icon={FileText}
  title="No documents found"
  description="Try adjusting your filters or upload a new document"
  action={{
    label: "Upload Document",
    onClick: () => router.push('/upload')
  }}
/>

// With secondary action
<EmptyState
  icon={FileText}
  title="No documents found"
  description="Try adjusting your filters or upload a new document"
  action={{
    label: "Upload Document",
    onClick: () => router.push('/upload')
  }}
  secondaryAction={{
    label: "Clear Filters",
    onClick: () => resetFilters()
  }}
/>
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `icon` | `LucideIcon` | Optional icon to display above the title |
| `title` | `string` | The main title of the empty state |
| `description` | `string` | A description of why there's no data and what to do |
| `action` | `{ label: string, onClick: () => void }` | Optional primary action button |
| `secondaryAction` | `{ label: string, onClick: () => void }` | Optional secondary action button |
| `className` | `string` | Optional additional CSS classes |

## Best Practices

1. **Be specific**: Clearly explain why there's no data and what the user can do about it.

2. **Provide actions**: When possible, include action buttons that help users resolve the empty state.

3. **Use appropriate icons**: Choose icons that represent the content type (e.g., FileText for documents, Package for loads).

4. **Maintain consistency**: Use the EmptyState component consistently across the application for a unified experience.

5. **Consider context**: Adapt the empty state message based on the context (e.g., "No documents found" vs. "No documents match your filters").
